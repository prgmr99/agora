# Agora — Vision & Roadmap

**"자율 AI 에이전트들을 위한 작업 조율 인프라"**

## 핵심 포지션

Claude Code, Codex, Cursor Agent, 커스텀 MCP 에이전트가 동시에 돌아갈 때,  
이들이 서로를 인식하고 태스크를 주고받을 수 있는 **중립적인 스위치보드**.

OpenClaw나 oh-my-claudecode와 경쟁하는 게 아니라,  
그 위에서 또는 내부에서 동작하는 **인프라 레이어**.

---

## 타겟 유저

**자율 AI 에이전트 2~5개를 동시에 운영하는 개발자**

- Claude Code 세션이 백엔드 작업 중
- Codex CLI가 프론트엔드 작업 중  
- 커스텀 MCP 에이전트가 테스트 실행 중
- → 이들이 서로를 인식하고 태스크를 넘길 방법이 없다. Agora가 그 스위치보드.

---

## 차별점

| 기존 도구 | 하는 것 | 못 하는 것 |
|---|---|---|
| MCP | 에이전트→툴 호출 | 에이전트→에이전트 위임 |
| A2A Protocol | 에이전트 간 메시지 | 태스크 스코어링, 큐, 재시도 |
| Claude Agent Teams | 동종 Claude 조율 | 크래시 복구, 세션 영속성 |
| LangGraph/CrewAI | 파이썬 워크플로우 | 런타임 동적 에이전트 등록 |

**Agora가 채우는 빈자리:**  
"현재 살아있는 에이전트 중 이 태스크에 가장 적합한 것에 신뢰성 있게 전달"

---

## 로드맵

### Phase 1: 신뢰성 (지금)
> "에이전트가 죽어도 태스크가 안 죽는다"

- **Task Lease + 크래시 복구** — heartbeat 기반 임대, 만료 시 자동 re-queue
- **`agora_renew_lease`** 툴 — 에이전트가 살아있음을 태스크 단위로 증명

### Phase 2: 스마트 라우팅
> "더 잘하는 에이전트에게 간다"

- 실행 텔레메트리 수집 (완료 시간, 재시도 횟수, 성공률)
- 텔레메트리 가중 라우팅 (keyword 매칭 + 이력 점수)
- 임베딩 기반 시맨틱 매칭 (키워드 한계 극복)

### Phase 3: 생태계 연결
> "어떤 AI 에이전트든 연결된다"

- A2A Agent Card 지원 (Google/Linux Foundation 표준)
- MCP ↔ A2A 브릿지
- 에이전트 레지스트리 (npm처럼 에이전트를 publish/discover)

---

## 성공 기준

- 에이전트가 크래시해도 30초 내 태스크 자동 복구
- Claude Code + Codex가 Agora를 통해 태스크를 주고받는 실제 데모
- A2A 호환 에이전트가 Agora에 자동 등록
